# Technology Stack & Data Structures

This document outlines the technology stack for each component of the AI Trading & Arbitrage Platform and defines the core data structures used for communication between agents.

## Technology Stack

| Agent              | Technologies                                                                                             | Rationale                                                                                                  |
| ------------------ | -------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| **Data Pipeline**    | Python (aiohttp, websockets), Google Pub/Sub, BigQuery, Cloud Storage                                    | High-performance async libraries for I/O, scalable messaging, and durable, cost-effective storage.         |
| **Features**       | Python, Apache Beam, Google Dataflow, Vertex AI Feature Store                                            | Scalable batch and stream processing for feature engineering, with a managed, low-latency feature store. |
| **Model Pipeline**   | Python, TensorFlow/PyTorch, Scikit-learn, Vertex AI Pipelines & Endpoints                                  | Industry-standard ML frameworks with a fully managed MLOps platform for reproducibility and deployment.    |
| **Execution Engine** | Go, gRPC                                                                                                 | High-performance, low-latency language ideal for the execution-critical path. gRPC for efficient comms.  |
| **API Gateway**    | FastAPI (Python), Google Cloud Endpoints, Firebase Authentication                                        | Fast, modern web framework with built-in API documentation, managed gateway, and secure user authentication. |
| **Frontend (Web)**   | Next.js (React), Tailwind CSS, TradingView Lightweight Charts                                            | A leading React framework for performance and SEO, utility-first CSS, and high-quality financial charts.   |
| **Frontend (Mobile)**| Flutter, Dart                                                                                            | Cross-platform framework for building high-quality native apps for iOS and Android from a single codebase. |
| **Infrastructure** | Terraform, Google Cloud Build, Google Kubernetes Engine (GKE), Docker                                    | Infrastructure-as-code for reproducibility, automated CI/CD, and scalable, containerized deployments.      |
| **Security**       | Google Secret Manager, Google Key Management Service (KMS), IAM                                          | Secure, centralized secret management, cryptographic key control, and fine-grained access policies.        |
| **QA & Ops**       | Pytest, Locust, Chaos Toolkit, Google Cloud Monitoring, Grafana                                          | Robust testing frameworks, performance testing, chaos engineering, and comprehensive observability.        |

## Core Data Structures (Protobuf)

The following data structures are defined in `contracts/trading.proto` and are used for messages passed between agents via Pub/Sub.

```protobuf
syntax = "proto3";

package trading;

import "google/protobuf/timestamp.proto";

// Enum for Order Side
enum Side {
  SIDE_UNSPECIFIED = 0;
  BUY = 1;
  SELL = 2;
}

// Enum for Order Type
enum OrderType {
  ORDER_TYPE_UNSPECIFIED = 0;
  MARKET = 1;
  LIMIT = 2;
}

// L2 Order Book Level
message OrderBookLevel {
  double price = 1;
  double quantity = 2;
}

// A complete L2 Order Book Snapshot
message OrderBook {
  string exchange = 1;
  string instrument = 2;
  google.protobuf.Timestamp timestamp = 3;
  repeated OrderBookLevel bids = 4;
  repeated OrderBookLevel asks = 5;
}

// A single market trade
message Trade {
  string exchange = 1;
  string instrument = 2;
  google.protobuf.Timestamp timestamp = 3;
  double price = 4;
  double quantity = 5;
  Side side = 6;
}

// A calculated feature vector
message Feature {
  string instrument = 1;
  google.protobuf.Timestamp timestamp = 2;
  double vwap_5m = 3;
  double orderflow_imbalance = 4;
  // ... other feature fields
}

// A trading signal generated by a model
message Signal {
  string strategy_id = 1;
  string instrument = 2;
  google.protobuf.Timestamp timestamp = 3;
  Side side = 4;
  double price_target = 5;
  double quantity = 6;
  OrderType order_type = 7;
}

// An order to be sent to an exchange
message Order {
  string order_id = 1;
  string client_order_id = 2;
  string exchange = 3;
  string instrument = 4;
  Side side = 5;
  double price = 6;
  double quantity = 7;
  OrderType type = 8;
  // ... other order fields
}

// The result of an order execution
message Execution {
  string execution_id = 1;
  string order_id = 2;
  string exchange = 3;
  string instrument = 4;
  google.protobuf.Timestamp timestamp = 5;
  double price = 6;
  double quantity = 7;
  double fees = 8;
  // ... other execution fields
}
```
